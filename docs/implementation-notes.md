# Implementation Notes — memo-app Bookmark Widget

> Spec: `docs/bookmark-widget-spec.md`  
> 実装開始: 2026-06-23

---

## Design Decisions

### [Bookmark Registration] 外部URLのD&D登録を廃止

**Context:** ブラウザURL欄・タブ・ブラウザUI由来のD&Dは、`window` / `document` / `document.body` の capture フェーズにもイベントが届かないケースが確認された。ページ内リンクのD&Dは反応するが、URL欄/タブD&Dを安定して拾えない。

**Decision:** ブックマーク登録手段としての外部URLドラッグ&ドロップは廃止する。登録はウィジェット内の追加ボタンから `BookmarkItemModal.vue` を開き、タイトル・説明・URLを入力して行う。ウィジェット内部のブックマーク並び替えD&D（vuedraggable）は残す。

**Rationale:** ブラウザ実装やドラッグ元によってWebページへイベント配送されないため、アプリ側の実装で安定動作を保証できない。中途半端に「一部リンクだけD&D可能」にするとユーザー体験が混乱するため、登録UIはモーダル入力に一本化する。

---

### [UI] 追加・編集はモーダル式に統一

**Context:** ブックマークはURLだけでなく表示文・説明も持つため、インライン入力よりモーダル入力の方が項目編集と共通化しやすい。

**Decision:** 追加ボタンで追加モーダルを開き、既存アイテムの✏️から同じモーダルを編集モードで開く。

**Rationale:** 追加と編集で同一UIを再利用でき、説明フィールドや将来の項目追加にも拡張しやすい。

---

## Deviations

### [Spec] 初期案の外部D&D登録をスコープ外へ変更

**Context:** 初期仕様ではブラウザURL欄・他タブ・ページ内リンクからのD&D登録を検討していた。

**Decision:** 実環境でURL欄/タブ由来のD&DイベントがWebページへ届かないことを確認したため、外部D&D登録を仕様から削除した。

**Rationale:** ページ内リンクだけ対応すると操作対象によって挙動が変わり、仕様として説明しづらい。安定した登録導線を優先する。

---

## Tradeoffs

### [UX] D&D登録を廃止し、追加ボタン入力に一本化

**Context:** D&D登録は成功すれば便利だが、ブラウザUI由来のD&Dが拾えない環境では失敗体験になる。

**Decision:** 登録は追加ボタンからのモーダル入力に限定する。

**Rationale:** 1クリック増えるが、入力・編集・説明追加の一貫性が高く、ブラウザ依存の不安定さを避けられる。
